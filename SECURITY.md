# 보안·권리 신고 안내 (Security & Privacy Reporting)

> 본 문서는 법률 자문이 아니며, 프로젝트 운영 원칙을 정리한 것입니다.

**R-punghoe(알풍회)** 는 정적(static) 사이트로 운영되는 비공식 팬 아카이브입니다.
서버·데이터베이스·백엔드가 없으며, 모든 기록은 공개 자료를 기준으로 합니다.

이 문서는 두 가지 신고 경로를 안내합니다.

1. **사이트 보안 취약점** 신고
2. **개인정보·권리 침해 / 정정·삭제(takedown)** 요청

---

## 1. 사이트 보안 취약점 신고

다음과 같은 기술적 문제를 발견하면 알려 주세요.

- 빌드·배포 파이프라인의 설정 오류
- 의존성 취약점, 잘못 노출된 토큰/시크릿
- XSS 등 정적 사이트에서 발생 가능한 클라이언트 측 문제
- 그 외 사이트 무결성에 영향을 줄 수 있는 문제

### 신고 방법 (권장 순서)

1. **GitHub Security Advisory(비공개 보고)**: 저장소 → **Security → Report a vulnerability**
   - <https://github.com/hjkwon-github/R-punghoe/security/advisories/new>
2. 위 기능을 사용할 수 없는 경우, **민감 정보가 없는 범위에서만** 일반 Issue로 알려 주세요.

### 책임 있는 공개(Responsible Disclosure)

- 문제가 수정되어 사용자가 보호될 때까지 **세부 내용 공개를 미뤄** 주세요.
- 실제 데이터 손상·서비스 중단·타인 정보 접근을 시도하지 말아 주세요.
- 선의의 신고자에 대해서는 비공개로 협력하며 감사를 표합니다.

---

## 2. 개인정보·권리 침해 / 정정·삭제 요청

기록 내용에 대한 다음 요청은 이 경로로 처리합니다.

- 사생활·식별정보(가족·주소·연락처·사적 동선 등) 노출
- 사실 오류 / 맥락 왜곡 / 명예 관련 우려
- 저작권·과도한 복제
- 본인 또는 정당한 권리자의 삭제·정정 요청

### 신고 방법

- **GitHub Issue** 의 `정정·삭제 요청(takedown-request)` 이슈 템플릿을 사용해 주세요.
  - 새 이슈 작성: <https://github.com/hjkwon-github/R-punghoe/issues/new/choose>
- **사생활·민감정보가 포함되어 공개 이슈로 남기기 곤란한 경우**, 위 1번의 비공개 보고(GitHub Security Advisory) 경로를 사용하셔도 됩니다. 공개 노출 없이 접수합니다.

### 요청 시 포함하면 좋은 정보

- 대상 페이지 URL과 해당 문장/항목
- 사유(사생활 / 오류 / 저작권 / 명예 등)
- 요청자 구분: **본인 / 정당한 권리자 / 제3자**
- 회신받을 연락 방법

> **대상 본인 또는 정당한 권리자의 요청은 최우선으로 처리합니다.**
> 사생활·민감정보 관련 요청은 검토 전이라도 우선 비공개(선조치) 처리할 수 있습니다.

자세한 검토·처리·기록 기준은 [콘텐츠·운영 정책(docs/policy.md)](docs/policy.md)의 "정정·삭제·이의제기 절차"를 참고하세요.

## 3. 응답 시점 (best-effort)

이 프로젝트는 자원봉사로 운영되는 비공식 팬 프로젝트입니다. 정해진 SLA는 없으며 **최선의 노력(best-effort)** 기준으로 응답합니다.

| 구분 | 1차 응답(접수 확인) 목표 |
| --- | --- |
| 사생활·민감정보 노출 (긴급) | 가능한 한 빠르게, 통상 수일 이내 |
| 저작권·명예·사실 오류 | 통상 1~2주 이내 |
| 일반 보안 취약점 | 통상 1~2주 이내 |

위 기간은 보장된 약속이 아니라 목표치입니다. 긴급한 사생활·안전 사안은 제목에 `[긴급]`을 붙여 주시면 우선 확인합니다.
