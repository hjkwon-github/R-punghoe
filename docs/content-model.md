# 콘텐츠 / 데이터 모델

이 문서는 R-punghoe(알풍회) 아카이브의 **기록(record) 데이터 모델**을 설명합니다.
모든 기록은 마크다운 파일 하나로 표현되며, 데이터베이스나 서버 없이 정적 사이트로만 동작합니다.

> 사이트 운영 원칙은 [기록 원칙](../src/content/docs/principles.md) 문서를 함께 참고하세요.
> 이 문서는 개발/편집자용 설계 문서이고, 사이트에 페이지로 노출되지 않습니다.

---

## 1. 한눈에 보기

- **기록 1개 = 마크다운 파일 1개** (`src/content/docs/archive/records/` 아래)
- 각 파일은 **프런트매터(frontmatter)** 로 구조화된 메타데이터를 담습니다.
- 스키마는 Starlight 기본 스키마를 확장해 정의합니다: `src/content.config.ts`
- 검색은 Starlight 내장 **Pagefind**(클라이언트 사이드)로 처리합니다. 별도 백엔드 없음.
- 사이드바의 "기록 목록"은 `archive/records` 디렉터리에서 **자동 생성**됩니다.

---

## 2. 스키마 (프런트매터 항목별 설명)

스키마는 `src/content.config.ts`에서 `docsSchema({ extend: z.object({ ... }) })`로
정의되며, **추가 항목은 모두 선택(optional)** 입니다. 따라서 기존 페이지(`about.md`,
`principles.md` 등)는 수정 없이 그대로 빌드됩니다.

| 항목 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `title` | 문자열 | 필수 | 페이지 제목. Starlight 기본 항목. |
| `description` | 문자열 | 권장 | 한 줄 요약. 검색·미리보기에 사용. Starlight 기본 항목. |
| `recordDate` | 날짜(YYYY-MM-DD) | 선택 | 발언/사건이 실제로 있었던 날짜. 연대기 정렬 기준. |
| `verification` | `확인됨` \| `검토중` \| `보류` | 선택 | 검증 상태. 아래 셋 중 하나만 허용. |
| `recordTags` | 문자열 배열 | 선택 | 분류 태그. (예약어 충돌을 피하려고 `tags`가 아닌 `recordTags` 사용) |
| `sources` | 객체 배열 | 선택 | 공개 출처 목록. 항목별 `title`, `url`, `timestamp?`. |
| `related` | 문자열 배열 | 선택 | 관련 기록의 파일명 또는 제목. |

### 2.1 `sources` 객체

각 출처는 다음 구조를 가집니다.

| 항목 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `title` | 문자열 | 필수 | 출처 제목(영상/기사/채널명 등) |
| `url` | URL | 필수 | 공개 링크. 누구나 접근 가능해야 함. |
| `timestamp` | 문자열 | 선택 | 영상/방송이라면 발언 등장 시각. 예: `1:23:45`, `12:30` |

> `url`은 zod의 URL 검증을 통과해야 합니다. 비공개·접근 불가 링크는 사용하지 않습니다.

### 2.2 왜 모두 선택 항목인가

`archive/records/` 밖의 일반 문서(소개, 원칙, 기여 등)도 같은 `docs` 컬렉션에 속합니다.
추가 항목을 필수로 만들면 그 문서들의 빌드가 깨집니다. 그래서 **모든 추가 항목은
선택**으로 두고, 기록 문서에서만 채워 넣는 규칙을 운영으로 강제합니다.

## 3. 검증 라이프사이클

검증 상태는 **정확히 세 가지** 값만 사용합니다. 진행 방향은 다음과 같습니다.

```
보류  ──>  검토중  ──>  확인됨
(held)    (under review)  (confirmed)
```

| 상태 | 의미 | 처리 |
| --- | --- | --- |
| `보류` | 출처가 불충분하거나 진위가 불명확함 | 본문 편입 보류. 더 나은 출처를 찾기 전까지 대기. |
| `검토중` | 출처는 있으나 맥락·정확도 확인이 더 필요함 | 공개는 하되 "검토중"임을 명시. |
| `확인됨` | 1차 공개 출처로 확인됨 | 정식 기록으로 인정. |

- 상태는 한 방향으로만 올라가는 것이 기본이지만, 새 정보로 신뢰가 떨어지면
  `확인됨 → 검토중`처럼 되돌릴 수도 있습니다. 되돌릴 때는 **검증 메모**에 사유를 남깁니다.
- 어떤 상태든 출처 없는 소문, 사생활, 투자 권유, 공격성 서술은 기록하지 않습니다.

## 4. 태그 분류(taxonomy)

`recordTags`는 자유 문자열이지만, 일관성을 위해 아래 **표준 분류**에서 고르는 것을 권장합니다.

### 4.1 형식(매체) 태그 — 1개 이상 권장

| 태그 | 사용 시점 |
| --- | --- |
| `영상` | 유튜브 업로드 영상에서 나온 내용 |
| `방송` | 라이브/스트리밍 방송에서 나온 내용 |
| `기사` | 기사·인터뷰 등 글 출처 |

### 4.2 주제(성격) 태그 — 필요한 만큼

| 태그 | 사용 시점 |
| --- | --- |
| `밈` | 팬덤 안에서 반복되는 별명·농담·짤 |
| `용어` | 반복적으로 쓰이는 표현·신조어 |
| `시장발언` | 거시경제·시장·차트 관점 발언(투자 권유 아님) |
| `자료언급` | 방송에서 언급된 책·보고서·데이터 등 |

> 태그는 위 목록을 우선 사용하고, 새 태그가 필요하면 이 문서에 먼저 추가한 뒤 사용합니다.
> 한 기록에 형식 태그 1개 + 주제 태그 1~3개 정도가 적당합니다.

## 5. 연대기(timeline.md)와의 관계

[연대기 문서](../src/content/docs/archive/timeline.md)는 개별 기록을 **날짜순으로 모아 보여 주는 색인**입니다.

- 각 기록의 `recordDate`가 연대기 정렬의 기준입니다.
- 연대기 항목은 짧게(날짜 · 제목 · 검증 상태 · 한 줄 요약 · 기록 링크)만 적고,
  자세한 내용은 개별 기록 페이지로 연결합니다.
- 같은 정보를 두 곳에 길게 중복하지 않습니다. **상세는 기록 파일, 색인은 연대기**가 원칙입니다.
- 정적 사이트이므로 연대기 갱신은 수작업(또는 추후 빌드 스크립트)으로 합니다.
  자동화는 "미래 아이디어"로만 두고, 지금은 DB·서버 없이 마크다운으로 유지합니다.

## 6. 파일 이름 규칙

기록 파일은 `src/content/docs/archive/records/` 아래에 둡니다.

```
YYYY-MM-DD-짧은-제목.md
```

- 예: `2026-05-12-macro-book-mention.md`, `2026-05-12-거시-책-언급.md`
- 날짜 접두사는 `recordDate`와 맞추면 파일 정렬과 연대기 관리가 쉬워집니다.
- 공백 대신 하이픈(`-`)을 쓰고, 슬러그가 곧 URL이 됩니다.
- **밑줄(`_`)로 시작하는 파일은 라우팅되지 않습니다.**
  - `_record-template.md`: 복사용 템플릿(페이지로 노출 안 됨)
  - 작성 중인 초안도 `_`를 붙여 두면 공개 전까지 라우팅을 막을 수 있습니다.
- 임시로 숨기고 싶다면 프런트매터에 `draft: true`를 써서 프로덕션 빌드에서 제외할 수도 있습니다.

---

## 7. 어떻게 "정적"으로 유지되는가

- 모든 기록은 **마크다운 파일**입니다. 데이터베이스가 없습니다.
- 스키마 검증은 **빌드 타임**에 zod로 수행됩니다(런타임 서버 없음).
- 검색은 빌드 시 생성되는 **Pagefind 인덱스**로 클라이언트에서 동작합니다.
- 사이트맵은 `site`가 설정되어 있어 Starlight가 자동으로 생성합니다(별도 통합 추가 불필요).
- 기여는 **GitHub Issue / PR로만** 받습니다.
- 데이터베이스, Workers, D1, R2, KV, 서버 함수 등 **백엔드는 일절 사용하지 않습니다.**
  필요해 보이는 기능이 있어도 "미래 아이디어"로만 남기고 정적 구조를 유지합니다.

## 8. 채워진 프런트매터 예시

```md
---
title: 라이브에서 언급한 거시경제 책
description: 알상무가 라이브에서 거시 입문서를 추천 도서로 언급한 기록
recordDate: 2026-05-12
verification: 검토중
recordTags:
  - 영상
  - 시장발언
  - 자료언급
sources:
  - title: '알상무 라이브 - 거시 흐름 이야기'
    url: https://www.youtube.com/watch?v=VIDEO_ID
    timestamp: '1:12:30'
related:
  - 2026-05-09-거시-관점-정리
---

# 라이브에서 언급한 거시경제 책

## 요약
...

## 맥락
...

## 출처
...

## 검증 메모
- 현재 상태: 검토중
- 추가 확인 필요: 정확한 발언 시각 재확인
```

실제 동작 예시는 사이트의 [예시 기록](../src/content/docs/archive/records/example-record.md)과
복사용 [템플릿](../src/content/docs/archive/records/_record-template.md)을 참고하세요.
