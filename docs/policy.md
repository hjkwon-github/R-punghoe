# 콘텐츠·운영 정책 (관리자용)

> 본 문서는 법률 자문이 아니며, 프로젝트 운영 원칙을 정리한 것입니다.

이 문서는 **R-punghoe(알풍회)** 의 기록을 검토·등록·관리하는 관리자(메인테이너)와 적극 기여자를 위한 운영 기준입니다.
알풍회는 실재하는 공인(알상무)의 **공개 활동**을 다루는 비공식 팬 아카이브이므로, 모든 결정은 "공개·검증·존중·저위험" 네 가지를 기준으로 내립니다.

일반 기여 안내는 [기여하기](../src/content/docs/contributing.md), 공개 면책 고지는 [면책·안내](../src/content/docs/disclaimer.md)를 참고하세요.

## 1. 공개정보 원칙

알풍회는 **누구나 같은 방법으로 다시 확인할 수 있는 공개 자료**만 기록합니다.

### 1.1 허용되는 출처

- 공개 유튜브 영상·라이브 다시보기 및 그 설명란
- 공개 기사, 보도자료, 공개 인터뷰
- 공식 채널(본인·소속 채널 등)의 공개 게시물
- 공개된 방송에서 본인이 직접 한 발언

### 1.2 출처 기록 방법 (모든 기록의 최소 요건)

하나의 기록에는 다음 메타데이터가 반드시 함께 있어야 합니다. 하나라도 비어 있으면 본문 편입 전 단계(`검토중` 또는 `보류`)로 둡니다.

| 항목 | 설명 | 예시 |
| --- | --- | --- |
| 날짜 | 발언/게시 시점(원본 기준) | `2025-04-12` |
| 출처 | 원본 공개 URL | `https://youtu.be/...` |
| 타임스탬프 | 영상 내 위치(가능하면 시작~끝) | `12:34` 또는 `12:34–13:10` |
| 검증 상태 | `확인됨` / `검토중` / `보류` 중 하나 | `검토중` |

- **출처가 사라질 수 있는 점**을 감안해, 가능하면 게시 일자·채널명·영상 제목 등 식별 정보를 함께 적습니다.
- **사적으로 공유된 자료(DM, 비공개 영상, 단톡방 캡처 등)는 출처로 인정하지 않습니다.** 공개되어 있더라도 출처가 불분명하면 등록하지 않습니다.
- 가공·편집된 2차 자료는 가능하면 1차 공개 원본을 추적해 함께 기록합니다.

## 2. 명예훼손 회피 가이드

공인의 공개 활동을 다루더라도, 표현 방식에 따라 명예를 훼손할 수 있습니다. 다음 원칙을 지킵니다.

### 2.1 사실과 의견을 구분합니다

- **사실 기술**: 출처·날짜·타임스탬프로 검증 가능한 내용만 단정적으로 적습니다.
  - 예: `2025-04-12 영상 12:34에서 "○○"라고 말함 (출처 링크).`
- **의견·해석**: 작성자의 해석임이 드러나게 적고, 사실처럼 보이지 않게 합니다.
  - 예: `…로 들린다`, `…라는 해석이 팬덤에서 있었다`처럼 출처와 주체를 밝힙니다.
- 인용은 **원문 그대로** 옮기되, 맥락을 왜곡하는 발췌(앞뒤 잘라내기)는 금지합니다.

### 2.2 검증되지 않은 주장은 상태로 표시합니다

- 사실 확인이 끝나지 않은 내용은 본문에서 단정하지 않고 `검토중` 또는 `보류`로 둡니다.
- 추측성 표현(`…일 것이다`, `사실은 …이다`)으로 확정 짓지 않습니다.
- 한 기록 안에서 확인된 부분과 미확인 부분이 섞여 있으면, 미확인 부분을 명시적으로 분리해 표시합니다.

### 2.3 단정적·비방 표현 금지

- 인신공격, 모욕적 별칭, 비하·조롱, 범죄·부도덕을 암시하는 단정 표현은 금지합니다.
- 부정적 평가가 필요한 경우에도 **출처가 있는 사실 + 중립적 서술**로만 적습니다.
- 알상무 본인뿐 아니라 영상에 등장하는 **제3자(다른 출연자·일반인)** 에 대해서도 동일하게 적용합니다.

## 3. 초상권 · 퍼블리시티 · 개인정보 보호

### 3.1 기록 금지 목록 (사적 영역)

다음은 공개되어 있는 것처럼 보이더라도 **수집·정리·연결하지 않습니다.**

- 가족·지인 등 사생활 관계 정보
- 집/회사 등 **주소·위치**, 실시간 동선, 출퇴근 경로
- 전화번호·이메일·메신저 ID 등 **연락처**
- 주민등록·금융·건강 등 민감정보
- 공개 활동과 무관한 **사적 일상**(개인 일정, 사적 모임 등)
- 여러 공개 조각을 모아 사생활을 **재구성·추론**하는 정리(프로파일링)

### 3.2 미성년·제3자 보호

- 영상에 우연히 등장한 일반인, 미성년자, 비공인은 **식별 가능한 형태로 기록하지 않습니다.**
- 필요한 경우 이름·얼굴 등 식별 정보를 제거하고 익명으로 처리합니다.
- 제3자에 대한 부정적 서술은 알상무 기록 목적에 꼭 필요한 최소한으로만 다룹니다.

### 3.3 초상·이미지 사용

- 인물 사진·캡처의 직접 게시는 지양하고, 가능하면 **원본 링크**로 대체합니다.
- 밈·자료로 이미지를 다룰 때도 비하·조롱 목적이 아니어야 하며, 저작권 기준(4장)을 함께 따릅니다.

## 4. 저작권

알풍회는 원저작물을 대체하지 않고 **가리키는(link to)** 아카이브를 지향합니다.

### 4.1 허용

- 짧은 인용(핵심 발언 한두 문장 수준)과 출처 표기
- 영상 내용의 **요약·정리**(작성자의 표현으로)
- 특정 시점을 가리키는 **타임스탬프 링크**
- 공식적으로 임베드가 허용된 공개 영상의 임베드

### 4.2 금지

- 영상 **전체 또는 상당 부분의 전사(스크립트 통째 복사)**
- 자막·대본·기사 본문의 과도한 전재
- 원본을 다시 볼 필요가 없을 정도의 **과도한 복제**
- 출처 표기 없는 콘텐츠 재게시

판단이 애매하면 "원본을 보러 가야 할 이유가 남아 있는가?"를 기준으로 삼습니다. 남아 있지 않다면 과도한 복제일 가능성이 큽니다.

## 5. 밈 처리 원칙

팬덤 밈은 문화적 기록 가치가 있으므로 **보존**합니다. 다만 다음을 지킵니다.

- 밈의 **유래·맥락·사용 시점**을 구분해 기록합니다(왜·언제·어떻게 쓰였는지).
- 특정 개인(알상무, 출연자, 일반인)을 **비하·공격·조롱**하는 용도의 밈은 보존하지 않습니다.
- 밈이 사실로 오해될 수 있으면 "밈/별칭"임을 명확히 표시합니다.
- 사생활을 소재로 한 밈은 3장의 금지 목록을 우선 적용합니다.

## 6. 정정·삭제·이의제기 절차 (correction / takedown)

오류·권리침해·사생활 우려가 제기되면 아래 절차로 처리합니다. **대상 본인 또는 정당한 권리자의 요청은 최우선으로 처리합니다.**

### 6.1 접수

- 접수 창구는 **GitHub Issue** 입니다. `정정·삭제 요청(takedown-request)` 이슈 템플릿을 사용합니다.
  - 새 이슈: <https://github.com/hjkwon-github/R-punghoe/issues/new/choose>
- 공개 이슈로 남기기 어려운 사생활·민감정보가 포함된 경우, [SECURITY.md](../SECURITY.md)의 비공개 신고 경로를 안내합니다.
- 접수 시 확인하는 정보: 대상 페이지/문장, 사유(오류·사생활·저작권·명예 등), 요청자 구분(본인/권리자/제3자), 연락 방법.

### 6.2 검토 기준

- **권리자 본인/정당한 권리자 요청**: 우선 검토하며, 사생활·민감정보·명예 관련 요청은 **선조치 후 검토**(우선 비공개 처리)할 수 있습니다.
- **사실 오류**: 출처 재확인 후 정정 또는 상태 강등(`확인됨`→`검토중`/`보류`).
- **저작권·과도 복제**: 4장 기준으로 축약·요약 대체 또는 삭제.
- **사생활·식별정보**: 3장 금지 목록에 해당하면 즉시 제거 검토.

### 6.3 처리 / 응답

가능한 조치(상황에 맞게 선택):

- 정정(내용 수정) / 출처 보강 / 상태 변경(`보류`로 강등) / 일부 또는 전체 삭제 / 비공개(히스토리 정리 포함).
- 요청자에게 처리 결과를 회신합니다. 응답 시점 기대치는 [SECURITY.md](../SECURITY.md)의 best-effort 기준을 따릅니다.

### 6.4 기록

- 모든 정정·삭제·이의제기 처리는 **무엇을·왜·어떻게** 처리했는지 해당 Issue/PR에 남깁니다.
- 단, 사생활·민감정보 보호를 위해 **처리 기록 자체가 침해를 재현하지 않도록** 구체 내용은 요약·비식별화해 남깁니다.

## 7. 검증 상태 운영 기준

기록은 항상 다음 세 상태 중 하나를 가집니다.

| 상태 | 의미 | 본문 노출 | 승격/강등 조건 |
| --- | --- | --- | --- |
| `확인됨` | 1차 공개 출처로 사실이 확인됨 | 본문 게시 | 출처·날짜·타임스탬프가 모두 검증됨 |
| `검토중` | 출처는 있으나 맥락/사실 확인 진행 중 | 제한적/명시 표시 | 확인 완료 시 `확인됨`으로 승격 |
| `보류` | 출처 불충분·권리 우려로 편입 전 | 본문 비노출 | 보강되면 `검토중`으로, 문제 있으면 유지/삭제 |

### 7.1 누가 상태를 바꿀 수 있나

- **신규 기록의 초기 상태**는 누구나 제안(PR/Issue)할 수 있으나, 기본값은 `검토중` 또는 `보류`입니다.
- **`확인됨`으로의 승격**은 **메인테이너(관리자)** 의 검토·머지를 통해서만 이루어집니다.
- **`보류`로의 강등 또는 삭제**는 권리침해·사생활 우려가 있을 때 메인테이너가 **즉시** 수행할 수 있습니다.
- 상태 변경 사유는 PR/커밋 또는 Issue에 간단히 남깁니다.

## 8. 법률 자문 아님 고지

> 본 문서는 법률 자문이 아니며, 프로젝트 운영 원칙을 정리한 것입니다.

이 정책은 알풍회를 윤리적이고 저위험으로 운영하기 위한 **자율 기준**입니다. 법적 판단이 필요한 사안은 자격 있는 전문가의 조언을 따라야 하며, 권리자의 정당한 요청이 있을 경우 이 정책과 무관하게 우선적으로 협조합니다.
