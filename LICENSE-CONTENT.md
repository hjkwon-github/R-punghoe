## 1. 적용 범위

이 라이선스(CC BY-SA 4.0)는 R-punghoe 저장소에 기여·정리된 **콘텐츠와 데이터**에 적용됩니다. 구체적으로 다음을 포함합니다.

- `src/content/` 아래의 위키 문서 (소개, 원칙, 연대기, 용어사전 등)
- 각 기록 항목의 본문, 요약, 분류(태그), 메타데이터(날짜·검증 상태 등 우리가 작성한 부분)
- 우리가 작성한 설명문, 정리 글, 번역·요약 텍스트

다음은 **이 라이선스가 적용되지 않습니다.**

- 저장소의 소스 코드 → [MIT 라이선스](./LICENSE)를 따릅니다.
- **제3자의 원저작물 자체** (예: 유튜브 영상, 기사 본문, 이미지, 음원 등). 이런 자료는 원저작권자에게 권리가 있으며, R-punghoe는 이를 재라이선스하지 않습니다. 우리는 이런 자료를 **링크·인용·짧은 요약**으로만 다루고, 원저작권은 각 저작권자에게 있습니다.
- 인물의 이름, 별명, 상표 등 식별 표지. 라이선스와 무관하게 인격권·상표권 등이 별도로 존재할 수 있습니다.

즉, **"우리가 직접 정리·작성한 텍스트"는 CC BY-SA 4.0**, **"원본 저작물"은 원저작권자의 권리**라는 점을 분명히 합니다.

### 허용되는 것

- **공유(Share)**: 어떤 매체나 형식으로든 복제·재배포할 수 있습니다.
- **변경(Adapt)**: 리믹스, 변형, 2차적 저작물 작성이 가능합니다. 상업적 목적도 포함됩니다.

### 지켜야 하는 조건

- **저작자표시 (BY)**: 적절한 출처를 표시하고, 라이선스 링크를 제공하며, 변경 사항이 있으면 그 사실을 밝혀야 합니다. 합리적인 방식이면 되지만, 이용 허락자가 이용자를 보증한다는 식으로 표시해서는 안 됩니다.
- **동일조건변경허락 (SA)**: 이 콘텐츠를 변형·각색한 결과물을 배포할 경우, **원본과 동일한 라이선스(CC BY-SA 4.0)** 로 배포해야 합니다.

## 4. 왜 라이선스를 둘로 나눴나요?

위키/아카이브 프로젝트는 성격이 다른 두 종류의 결과물을 만들어냅니다.

1. **사이트를 굴리는 코드** — 다른 사람이 비슷한 아카이브를 만들 때 부담 없이 재사용할 수 있도록 가장 관대한 **MIT**를 택했습니다.
2. **공동으로 정리한 기록(콘텐츠)** — 여러 기여자가 시간을 들여 모은 공공재 성격의 자료입니다. 누구나 가져다 쓰되, **출처를 남기고(저작자표시)**, **다시 공유할 때도 같은 자유를 유지(동일조건변경허락)** 하도록 **CC BY-SA 4.0**을 택했습니다. 이는 Wikipedia 등 다수의 공개 지식 아카이브가 사용하는 방식과 같은 취지입니다.

소프트웨어 라이선스(MIT)는 산문·데이터에는 잘 맞지 않고, 콘텐츠 라이선스(CC)는 코드에 적합하지 않기 때문에 둘을 분리하는 것이 표준적인 모범 사례입니다.

## 5. 기여 시 동의 사항

이 저장소에 콘텐츠를 기여(Issue, Pull Request 등)하면, 기여자는 다음에 동의하는 것으로 간주합니다.

- 본인이 작성·정리한 텍스트를 **CC BY-SA 4.0**으로 배포하는 데 동의합니다.
- 코드 변경분은 **MIT**로 배포하는 데 동의합니다.
- 제3자의 저작물을 그대로 옮기지 않으며, 인용·요약은 [기여 가이드](./CONTRIBUTING.md)와 운영 정책의 범위 안에서만 한다는 점을 확인합니다.

권리 침해나 삭제 요청이 있는 경우 GitHub Issue의 **삭제·정정 요청(takedown)** 템플릿을 통해 접수해 주세요. 운영 정책에 따라 신속하고 정중하게 처리합니다.
